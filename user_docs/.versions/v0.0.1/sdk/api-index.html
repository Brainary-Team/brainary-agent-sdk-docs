接口索引 ​

读完本页你能：按 Functions / Classes / Types 分组,一眼查到每个公开接口的实现状态与文档落点。

这是一张按接口名反查文档位置的索引,按 Functions / Classes / Types 分组。它与本章另两张表职责不同、互不重复:

  • 总览的接口一览 —— 接口的命名与状态;
  • 边界的覆盖矩阵 —— 每个接口的裁剪决策与理由;
  • 本页 —— Brainary 接口名 → 文档落点(开发者查阅用)。

状态说明：brainary-agent-sdk 当前全部接口均为 🟠【规划中，未完成】——架构已规划、尚未实现；下方给出的类型、签名、字段均为已承诺的形态，供对齐讨论，非占位草案。（⚪ = 暂缓 / 不纳入，非交付接口。）

Functions ​

接口状态一句话去哪页
query(prompt, options)🟠一次性入口,返回 'static 的 MessageStream两个入口
#[tool]（宏）🟠把一个类型标成工具自定义工具
FunctionTools::new(name, desc)🟠一组工具打包成命名能力单元自定义工具
内置工具目录（read_file / write_file / edit_file / find_file / grep 等）🟠（含 schema 精化缺口）随会话默认可用的一组内建工具；bash / web_fetch / web_search / ask_user_question / agent(🟠) 亦在此登记内置工具目录
list_sessions()🟠列举跨进程历史会话（返回形态已定,签名已定）会话管理
get_session_messages(id)🟠取某会话的消息（返回 TranscriptMessage 列表,形态已定）会话管理
get_session_info(id)🟠取某会话的元信息（返回 SessionInfo,形态已定）会话管理
rename_session(id, name)🟠重命名会话（签名已定）会话管理
tag_session(id, tag)⚪打标签,暂缓会话管理

Classes ​

接口状态一句话去哪页

Types ​

接口状态一句话去哪页
Options（Options::builder())🟠唯一配置对象,所有行为收敛于此Options
ModelSelection🟠模型接入信息（from_env() / 显式构造)Options
SystemPrompt🟠系统提示三来源（Default / Inline / TemplateFile)Options
Message（enum)🟠流的元素：Assistant / User / System / Result消息模型
AssistantMessage / UserMessage / SystemMessage / ResultMessage🟠四类消息（UserMessage 待上游填充)消息模型
ContentBlock（enum)🟠内容块：Text(🟠) / ToolUse / ToolResult / Thinking(🟠 类型就位不填充)消息模型
StopReason🟠终局原因四变体（Stopped / MaxTurns / Interrupted / ContextExhausted)消息模型
Usage / ResultMessage.total_cost_usd🟠成本面,类型就位、值恒 None 待上游成本
InterruptHandle🟠中断句柄（interrupt() / is_interrupted())中断与护栏
SessionTranscript / TranscriptMessage🟠只读会话快照（可序列化,不可续跑)会话导出
SessionInfo🟠会话元信息（形态已定)会话管理
PermissionMode（6 模） / CanUseTool / ToolPermissionContext / PermissionResult / PermissionUpdate / PermissionRuleValue🟠权限控制面全套：模式枚举（default / accept_edits / bypass_permissions / plan / … 6 态）、回调、上下文、判定结果、运行时增量更新与规则值权限模型
allowed_tools / disallowed_tools🟠工具准入白/黑名单（Options 字段,与权限判定联动)权限模型
agents / AgentDefinition🟠SDK 直挂子代理（在 Options 内声明式定义）Options · 子代理
setting_sources / add_dirs / sandbox / plugins / output_format / thinking / effort / fallback_model🟠Options 配置旋钮：设置来源、附加工作目录、沙箱、插件、输出格式、思考开关、努力档位、回退模型Options
HookEvent / HookMatcher / HookCallback / HookInput / HookJSONOutput🟠Hooks 子系统五件套：事件枚举、匹配器、回调、输入、结构化输出Hooks
AssistantMessageError🟠助手消息内的错误块（类型就位）消息模型
RateLimitEvent / RateLimitInfo🟠限流事件与配额信息（形态已定,未透出）消息模型
ToolAnnotations🟠工具行为提示（占位)自定义工具
McpServerConfig 家族（Stdio / SSE / Http + 鉴权 headers）🟠MCP server 富配置：三种传输 + 鉴权头（糖仍可只收 URL）MCP 接入
BrainaryError / ErrorCategory🟠一个 #[non_exhaustive] 枚举 + category() 可恢复性分类（家族目录在 core)；不含 CLI*Error 一族错误处理
PoaOutcome / RunReport🟠PoA 运行结果与报告（poa feature)运行一个 PoA
