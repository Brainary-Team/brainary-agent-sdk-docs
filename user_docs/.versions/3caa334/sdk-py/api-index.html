接口索引（Python） ​

读完本页你能：按 Functions / Classes / Types 分组,一眼查到每个接口的规划状态与文档落点。

这是一张按接口名反查文档位置的索引,按 Functions / Classes / Types 分组。它与本章另两张表职责不同、互不重复:

  • 总览的接口一览 —— 接口的命名与状态;
  • 边界的覆盖矩阵 —— 每个接口的裁剪决策与理由;
  • 本页 —— Brainary 接口名 → 文档落点(开发者查阅用)。

本章整体是「规划：镜像 Rust 的 Python 接口面」

下表状态标记统一为 🟠【规划中，未完成】；Rust 与 Python 两侧当前均尚未实现,签名为设计形态。详见 总览 规划横幅。

状态说明：brainary-agent-sdk 当前全部接口均为 🟠【规划中，未完成】——架构已规划、尚未实现；下方给出的类型、签名、字段均为已承诺的形态，供对齐讨论，非占位草案。（⚪ = 暂缓 / 不纳入，非交付接口。）

Functions ​

接口状态一句话去哪页
query(prompt, options)🟠一次性入口,直接是 AsyncIterator[Message],async for 消费两个入口
@tool(name, desc, schema)🟠把一个 async 函数标成工具自定义工具
create_sdk_mcp_server(name, version, tools)🟠一组工具打包成命名能力单元自定义工具
内置工具目录（read_file / write_file / edit_file / find_file / grep 等）🟠（含 schema 精化缺口）随会话默认可用的一组内建工具；bash / web_fetch / web_search / ask_user_question / agent(🟠) 亦在此登记内置工具目录
list_sessions()🟠列举跨进程历史会话（返回形态已定,签名已定）会话管理
get_session_messages(session_id)🟠取某会话的消息（返回 TranscriptMessage 列表,形态已定）会话管理
get_session_info(session_id)🟠取某会话的元信息（返回 SessionInfo,形态已定）会话管理
rename_session(session_id, title)🟠重命名会话（签名已定）会话管理
tag_session(session_id, tag)⚪打标签,暂缓会话管理

接口状态一句话去哪页
BrainaryClient🟠有状态多轮客户端；async with + query / receive_response / await interrupt() 及只读检视方法两个入口

Types ​

接口状态一句话去哪页
Options（dataclass)🟠唯一配置对象,所有行为收敛于此Options
ModelSelection🟠模型接入信息（Options.model_from_env() / 显式构造)Options
SystemPrompt🟠系统提示三来源（Default / Inline / TemplateFile)Options
Message（union)🟠流的元素,用 isinstance 判型消息模型
AssistantMessage / UserMessage / SystemMessage / ResultMessage🟠四类消息（UserMessage 待上游填充)消息模型
ContentBlock（union)🟠内容块：TextBlock(🟠) / ToolUseBlock / ToolResultBlock / ThinkingBlock(🟠 类型就位不填充)消息模型
stop_reason（4 取值)🟠终局原因（stopped / max_turns / interrupted / context_exhausted)消息模型
Usage / ResultMessage.total_cost_usd🟠成本面,类型就位、值恒 None 待上游成本
SdkMcpTool / CapabilityUnit🟠@tool 产物 / create_sdk_mcp_server() 产物自定义工具
SessionTranscript / TranscriptMessage🟠只读会话快照（export_transcript(),不可续跑)两个入口
SessionInfo / SessionMessage🟠会话元信息 / 会话消息（形态已定)会话管理
PermissionMode（6 模） / CanUseTool / ToolPermissionContext / PermissionResultAllow / PermissionResultDeny / PermissionUpdate / PermissionRuleValue🟠权限控制面全套：模式枚举（default / acceptEdits / bypassPermissions / plan / … 6 态）、回调、上下文、允/拒判定结果、运行时增量更新与规则值权限模型
allowed_tools / disallowed_tools🟠工具准入白/黑名单（Options 字段,与权限判定联动)权限模型
agents / AgentDefinition🟠SDK 直挂子代理（在 Options 内声明式定义）Options · 子代理
setting_sources / add_dirs / sandbox / plugins / output_format / thinking / effort / fallback_model🟠Options 配置旋钮：设置来源、附加工作目录、沙箱、插件、输出格式、思考开关、努力档位、回退模型Options
HookEvent / HookMatcher / HookCallback / HookInput / HookJSONOutput🟠Hooks 子系统五件套：事件枚举、匹配器、回调、输入、结构化输出Hooks
AssistantMessageError🟠助手消息内的错误块（类型就位）消息模型
RateLimitEvent / RateLimitInfo🟠限流事件与配额信息（形态已定,未透出）消息模型
ToolAnnotations🟠工具行为提示（占位)自定义工具
McpServerConfig 家族（Stdio / SSE / Http + 鉴权 headers）🟠MCP server 富配置：三种传输 + 鉴权头（糖仍可只收 URL）MCP 接入
BrainaryError 异常类树（BuildError / AuthenticationError / RateLimitError / ToolExecutionError …）🟠自解释命名的异常子类,except 具体子类 分支;except BrainaryError 兜底。不设 CLI*Error 一族错误处理

上表已登记本轮新增的内置工具、Hooks、权限全套、Options 旋钮与消息新增类型。其余全表面（含 Transport / StreamEvent / 后台任务消息等尚未落地的接口)的处置决策见 边界与路线图 · 覆盖矩阵。
